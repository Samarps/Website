With the helps of Hooks, Jenkins, Plugins & Remote Trigger: once the Developer changed the code and commits:-
  • The Code got pushed to Github (with the help of Hooks)
  • Developer alerts Jenkins about new code being pushed and it needs to start the Job. (again using Hooks)
  • Jenkins goes to Github(any VCS) and clones the code & deploys on the webserver.
  • Now the new Website is available to clients


All these above steps are automated and agile.
Note:- One of best method that I have learned till now beacuse it is not limited to any SCM
- As Github Hook Trigger was specifically for Github.
